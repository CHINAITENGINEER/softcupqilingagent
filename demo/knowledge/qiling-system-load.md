# QilingOS 系统负载巡检知识

## 指标解释

系统负载反映运行队列和不可中断等待任务数量。负载需要结合 CPU 核心数、IO 等待、内存和进程状态判断。

## 只读巡检建议

1. 查看负载趋势。
2. 查看 CPU、内存、IO 等关键资源。
3. 查看占用最高的进程。
4. 如果发现异常，先输出诊断建议，不直接执行破坏性操作。

## 常见命令

```bash
uptime
cat /proc/loadavg
top -b -n 1 | head -n 20
free -h
df -h
iostat -xz 1 3
```

## 风险边界

- 读取系统负载属于低风险只读操作。
- kill 进程、清理目录、重启服务属于高风险操作。
- 高风险操作必须审批后执行。

## RAG 展示点

当用户输入“检查系统负载”时，Agent 可以检索该知识，解释负载指标，并给出下一步只读检查建议。若后续涉及重启或 kill，应进入审批闭环。
